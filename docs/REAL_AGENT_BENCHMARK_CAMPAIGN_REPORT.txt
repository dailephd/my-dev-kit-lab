Correctness summary:
- Main campaign average correctness delta: -0.1130125
- The aggregate answer remained "preserve correctness" because every paired comparison kept the same pass/fail outcome
- Claude generally answered more of the structured task than Codex, but my-dev-kit-guided correctness was still lower than raw-full-file in the completed Claude pairs

Interpretation of negative token savings:
- Negative token savings does not mean the experiment runner is broken
- It means the guided strategy produced larger or more verbose sessions than the raw-full-file strategy for the comparable run pair
- In this campaign the guided prompts and retrieval-oriented output contract often increased interaction overhead instead of shrinking it

Limitations:
- This is a partial real-agent campaign because Codex produced timeouts, invalid outputs, and limit-reached outcomes
- Claude did not expose token totals in these runs, so Claude token comparisons are unavailable
- Correctness is deterministic and answer-key based; there is no semantic LLM judge
- The report compares local CLI behavior, not cloud billing telemetry
- Visualization demos used the fake my-dev-kit CLI because the request focused on real agents, not real retrieval engine runs

Recommended next benchmark campaign:
- Keep these medium and large projects
- Add one more medium or large case per project so answer-key coverage is broader than a single retrieval task each
- Run a smaller Codex-only retry campaign with one complexity level at a time to separate timeout behavior from prompt-size effects
- Add a second mixed-language case that targets different Python and TypeScript modules to test whether guided retrieval helps on cross-language symbol navigation
